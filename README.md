# Breast EIT Neural Network Post-Processing

MATLAB code accompanying the manuscript:

> S. O. Aboagye, J. A. Hunt, and Y. Wei,  
> **"Resolving the Position-Size Trade-Off in Breast EIT Through Neural Network Post-Processing of 3D Reconstructions"**,  
> *IEEE Transactions on Biomedical Engineering*, [Under review, 2026].

This repository contains the full MATLAB pipeline used in the study, including the 3D forward model, the regularised reconstruction baselines, the three neural network training strategies, and the evaluation scripts. Running the code reproduces the figures and tables reported in the manuscript.

---

## What this code does

The code implements a complete simulation-and-evaluation pipeline for tumour parameter estimation from 3D breast electrical impedance tomography (EIT) data. The pipeline has four stages:

1. **Data generation.** Builds a 3D spherical-cap breast model with 16 electrodes in three concentric rings, places spherical tumour inclusions across a defined parameter space, and simulates boundary voltage measurements using EIDORS under the complete electrode model.
2. **Reconstruction.** Solves the regularised inverse problem using three priors (NOSER, Tikhonov, Laplace) and extracts tumour parameters using two threshold methods (fixed 50%, Otsu).
3. **Neural network training.** Trains three feedforward networks: Approach A learns directly from boundary voltages, Approach B post-processes PCA-compressed NOSER reconstructions, and Approach C concatenates both inputs.
4. **Evaluation.** Compares all nine methods on an 864-sample test set using paired Wilcoxon signed-rank tests with Bonferroni correction, generates the manuscript figures, and produces the performance tables.

## Quick start

The fastest path to a working pipeline is below. Allow about two hours for the full dataset to be generated on a modern desktop.

```matlab
% 1. Open MATLAB and navigate to the repository root
cd path/to/breast-eit-nn-postprocessing

% 2. Add all subfolders to the MATLAB path
setup_paths

% 3. Verify EIDORS is installed and on the path
% (run startup.m from your EIDORS install directory if needed)

% 4. Generate the training dataset (5,760 samples)
generate_training_data

% 5. Run all three neural network training routines
train_approach_A_direct
train_approach_B_postproc
train_approach_C_hybrid

% 6. Evaluate all methods on the test set
evaluate_all_methods

% 7. Generate the manuscript figures
generate_figures
generate_visual_comparison
```

## Configuration

All key parameters (mesh resolution, tumour ranges, noise levels, training hyperparameters) are centralised in `data_generation/dataset_config.m` and the head of each training script. To explore alternative parameter values, edit those files rather than hunting through the codebase.

## Troubleshooting

**EIDORS errors about Jacobian dimensions:**  
Make sure you have run EIDORS's `startup.m` for your version. The code expects a 208 × 19,403 Jacobian for the default mesh.

**Out-of-memory during PCA:**  
The full reconstruction matrix is 4,032 × 19,403 in single precision. If your machine has less than 16 GB of RAM, reduce the training set size in `dataset_config.m` or use the streaming PCA option.

**Network training fails to converge:**  
The Adam optimiser settings are conservative. If a particular run diverges, reduce the initial learning rate from 1e-3 to 5e-4 and re-run.
